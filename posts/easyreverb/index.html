混响 DSP 的实现

Feedback Delay Loop

在现实中，一个声音在房间中反复反弹扩散形成的回声和梳状滤波，这是我们辨别混响重要的特征，这幅图就体现了一种简易的回声发声器。

信号进入系统之后，先通过 delay 进行时间上的延迟，一部分信号直接输出，另一部分则是进入一个衰减的乘法器中再通过延迟输出，这样每次延迟输出的信号能量都比前一次小。

由此对单个信号我们有了这样一种让它在时间上进行扩散的结构，但是听起来更像是具有金属感的声音（梳状滤波），所以我们需要把上述结构拓展成多通道的。

现在我们有 multi channel 的信号进入这个系统，同样还是经过 delay，但是这次对每个通道的 delay time 都不一样。每个通道的信号经过衰减之后进入一个混合矩阵，这个矩阵的作用是把每个通道的信号都分发给所有通道，比如 channel 1 的信号进入矩阵后，通道 1 的回声会被按比例分配给通道 1、2、3、4 的输入。

为了保证这个矩阵不会无限发散所有信号，我们需要使用正交矩阵，保证前后的信号能量相一致。

现在我们的系统在时间上已经能够不错的进行扩散和持续，但是听起来还是只是很多信号的衰减延迟，没有把声音打碎成粒子的云雾感。

所以我们专门增加一步来先把信号进行扩散，再进入先前提到的 feedback loop 中。

多通道的信号在经过不同时间的 delay 后，对某些通道的信号进行反相，来打破信号的相干性。这一步还可能交换不同通道的信号，增加信号的复杂度。最后经过一个正交矩阵，把所有通道的信号平均输出到四个通道中。

最后我们把两个系统整合在一起，信号可能经过很多步的扩散，最后进入 feedback loop。

算法实现

Diffusion Step

template <int channels = 8>
struct DiffusionStep {
  using Array = std::array<double, channels>;
  double delayMsRange = 20;

  std::array<int, channels> delaySamples;
  std::array<Delay, channels> delays;
  std::array<bool, channels> flipPolarity;

  void configure(double sampleRate) {
    double delaySamplesRange = delayMsRange * 0.001 * sampleRate;
    for (int c = 0; c < channels; ++c) {
      double rangeLow = delaySamplesRange * c / channels;
      double rangeHigh = delaySamplesRange * (c + 1) / channels;
      delaySamples[c] = randomInRange(rangeLow, rangeHigh);
      delays[c].resize(delaySamples[c] + 1);
      delays[c].reset();
      flipPolarity[c] = rand() % 2;
    }
  }

  Array process(Array input) {
    // Delay
    Array delayed;
    for (int c = 0; c < channels; ++c) {
      delays[c].write(input[c]);
      delayed[c] = delays[c].read(delaySamples[c]);
    }

    // Mix with a Hadamard matrix
    Array mixed = delayed;
    Hadamard<double, channels>::inPlace(mixed.data());

    // Flip some polarities
    for (int c = 0; c < channels; ++c) {
      if (flipPolarity[c])
        mixed[c] *= -1;
    }

    return mixed;
  }
};

DiffusionStep 的代码主要分为两个部分：configure 和 process，方便我们在 PluginProcessor 里面的 PrepareToPlay() 和 ProcessBlock() 里进行调用。

代码中的 Delay 类主要使用了 Signalsmith Audio 的 DSP 库，大部分代码也是学习自这个 DSP 库作者在 ADC 上的演讲 Let’s Write a Reverb - Geraint Luff - ADC21，可以说是我看过最清晰的 DSP 小白向讲解了。

在 process() 中我们可以清楚地看到之前所说的三个用于 diffusion 的步骤：delay、mix matrix 和 flip polarity。

Feedback Loop

template <int channels = 8>
struct MultiChannelMixedFeedback {
  using Array = std::array<double, channels>;
  double delayMs = 50;
  double decayGain = 0.85;

  std::array<int, channels> delaySamples;
  std::array<Delay, channels> delays;

  void configure(double sampleRate) {
    double delaySamplesBase = delayMs * 0.001 * sampleRate;
    for (int c = 0; c < channels; ++c) {
      double r = c * 1.0 / channels;
      delaySamples[c] = std::pow(2, r) * delaySamplesBase;
      delays[c].resize(delaySamples[c] + 1);
      delays[c].reset();
    }
  }

  Array process(Array input) {
    Array delayed;
    for (int c = 0; c < channels; ++c) {
      delayed[c] = delays[c].read(delaySamples[c]);
    }

    // Mix using a Householder matrix
    Array mixed = delayed;
    Householder<double, channels>::inPlace(mixed.data());

    for (int c = 0; c < channels; ++c) {
      double sum = input[c] + mixed[c] * decayGain;
      delays[c].write(sum);
    }

    return delayed;
  }
};

这段代码的逻辑就更简单了，只是把 input 的信号 delay 后再 input 进去就可以实现衰减反馈了。

JUCE 的这行代码可以防止极小值带来的性能问题：juce::ScopedNoDenormals noDenormals;

参数的设置和绑定

JUCE 中参数的信号链

mix 参数的完整路径：

  1. PluginProcessor 构造时创建 APVTS，并注册 mix 参数
  2. Host（DAW/Standalone）读取到该参数（可自动化、可保存）
  3. PluginEditor 创建 Slider，用 SliderAttachment 绑定 “mix”
  4. 旋钮变化 -> 参数值进入 APVTS
  5. processBlock 从 APVTS 原子读取 mix，用于音频计算
  6. getStateInformation/setStateInformation 负责保存与恢复参数状态

PluginProcessor.h：声明参数

juce::AudioProcessorValueTreeState apvts;

  • 作用：参数容器 + 状态树 + 线程桥接。
  • 含义：所有参数（mix）都放在这里，供 GUI、Host、DSP 共用。

static ParameterLayout createParameterLayout();

  • 作用：集中定义参数列表（ID、范围、默认值、显示文本逻辑）。
  • 何时调用：在构造函数初始化 apvts 时调用。

juce::AudioProcessorValueTreeState& getAPVTS();

  • 作用：给 Editor 暴露参数系统，用于创建 attachment。

PluginProcessor.cpp：参数创建、使用、保存

A. 构造函数列表中实例化 apvts： apvts(*this, nullptr, "Parameters", createParameterLayout()) 四个参数分别是：

  1. this：把 APVTS 绑定到当前 AudioProcessor
  2. nullptr：不使用旧式 UndoManager（常见做法）
  3. “Parameters”：根 ValueTree 名称（状态树根节点）
  4. createParameterLayout()：参数定义集合 结果：mix 参数注册完成，Host 可见。

B. createParameterLayout()

juce::AudioProcessorValueTreeState::ParameterLayout
PluginProcessor::createParameterLayout() {
  std::vector<std::unique_ptr<juce::RangedAudioParameter>> parameters;
  parameters.push_back(std::make_unique<juce::AudioParameterFloat>(
      juce::ParameterID{"mix", 1}, "Mix",
      juce::NormalisableRange<float>(0.0f, 1.0f, 0.001f), 0.25f, juce::String(),
      juce::AudioProcessorParameter::genericParameter,
      [](float value, int) {
        return juce::String(juce::roundToInt(value * 100.0f)) + "%";
      },
      [](const juce::String& text) {
        return juce::jlimit(
            0.0f, 1.0f,
            text.upToFirstOccurrenceOf("%", false, false).getFloatValue() /
                100.0f);
      }));
  return {parameters.begin(), parameters.end()};
}

AudioParameterFloat(ParameterID{"mix", 1}, "Mix", range, default, ...)

  • ParameterID{“mix”, 1}
  • “mix”：参数唯一 ID
  • 1：参数版本号（用于后续参数迁移兼容）
  • “Mix”：给 Host/GUI 展示的人类可读名字
  • NormalisableRange<float>(0.0f, 1.0f, 0.001f)：范围和步进
  • 0.25f：默认值
  • valueToText lambda：内部 0~1 显示成百分比文本
  • textToValue lambda：把 “25%” 解析回 0.25

C. processBlock(…)中读取参数 apvts.getRawParameterValue("mix")->load()

  • getRawParameterValue(“mix”) 返回 std::atomic<float>
  • load() 是实时线程安全读取（音频线程可用） 每个 block 读一次，再用于该 block 的样本循环

D. 状态保存与恢复 getStateInformation(...)

  • apvts.copyState() 取出当前参数状态树
  • createXml() 转 XML
  • copyXmlToBinary(…) 写给 Host 保存 setStateInformation(...)
  • getXmlFromBinary(…) 读回 XML
  • 校验 tag 后 apvts.replaceState(…) 恢复参数

PluginEditor.h/.cpp：参数可视化与交互

A. 组件定义（PluginEditor.h）

  • juce::Slider mixSlider;
  • std::unique_ptr<SliderAttachment> mixAttachment; SliderAttachment 类型是： juce::AudioProcessorValueTreeState::SliderAttachment

B. 构造函数绑定（PluginEditor.cpp） mixAttachment = std::make_unique<SliderAttachment>(processorRef.getAPVTS(), "mix", mixSlider); 完成双向同步：

  1. 参数变动 -> 旋钮位置自动更新
  2. 旋钮拖动 -> APVTS 里的 “mix” 自动更新

实际调用顺序

  1. 插件实例化
  • 调 PluginProcessor::PluginProcessor()
  • APVTS + mix 创建并注册到 Host
  1. 打开 GUI
  • 调 createEditor() -> PluginEditor 构造
  • 建立 SliderAttachment
  1. 用户转旋钮
  • mixSlider 值变
  • attachment 写入 APVTS 的 “mix”
  1. 音频回调
  • 周期性调 processBlock
  • 原子读取 “mix”，作用于音频
  1. 保存工程
  • Host 调 getStateInformation，存参数树
  1. 重开工程
  • Host 调 setStateInformation，还原参数树
  • attachment 使 GUI 自动显示恢复后的值

小结

至此我们完成了一个简单但效果还不错的 reverb，所有代码和一个简易的演示 VST 都在这个仓库：oreru/EasyEffect: a simple project trying juce for plugin。跑通整个流程之后，发现 DSP、插件这些领域也终于没有那么黑盒了，但想自己从零把原理到代码手搓出来还是很有难度。好在摸清原理后能够明白改动哪些代码会影响什么东西，之后可能会从这方面去探索插件和音频编程的可能性。
