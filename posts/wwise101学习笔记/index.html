发出第一个声音

创建event

  • 用于wwise捕捉输入的游戏调用
  • layout
  • designer
  • event
  • workunit
  • create event

导入声音

  • 在audio标签下找到Actor-Mixer Hierarchy
  • 我们需要将声音放在 Actor-Mixer Hierarchy（Actor-Mixer 层级结构）下的 Work Unit（工作单元）中。
  • 添加SoundFX节点即可
  • 右键导入音频文件

也可以直接将声音拖入work unit

使用action将event和SFX对象连接

  • 左下角event viewer里面选中之前的event
  • add一个action
  • 将SFX对象拖入target即可

把 Event 添加到 SoundBank 中去

**SoundBank 包含游戏当中（即运行时）所要使用的代码和音频素材。

  • 换成soundbank layout之后会出现soundbank manager
  • 直接new一个soundbank，命名需跟游戏一致
  • 把event拖入即可

生成SoundBank

  • 首先将soundbank的路径改为游戏需要的文件夹下
  • 然后记得勾选平台和语言
  • generate selected即可

设计声景

一声多用

  • 选中sound sfx对象时，可在右侧的sound sfx窗口中更改音高等属性
  • 从而达到一声多用

使用Source Editor

  • 更进一步的分离区分两个声音
  • 可以选取声音的起点和终点，是对source不同部分的引用

让单个event触发多个action

  • 选中对应的event之后直接将sfx对象拖入当前event窗口，就会自动创建play action
  • **注意上一步要先按住shift，保证选取对象的时候不会打开property窗口
  • 我们可以更改每个action对应的参数
  • 从而达到延迟播放等效果

使用音频随机化

  • **这里文档中提到了一个不同的导入方式：直接将素材所在文件夹拖至对应work unit上，然后更改文件夹属性为random container

控制随机化的参数

  • 在右侧的property editor中，有几个参数可以注意一下
  • random type ：选取shuffle模式时只有播放完容器中每个声音之后才会开始下一轮，避免重复
  • limitation ：限制重复播放某个对象之前必须播放的对象个数
基本每个参数都会附有一个randomizer，双击框就可以进行randomizer的参数调节

序列容器

  • 现在有一个情形：分别两个random container，一个负责宝石落地的头音，一个负责宝石破碎的尾音，如何每次按顺序只播放一次头音和尾音？
  • 首先需要在random container中将play mode改为step
  • 为这两个container创建一个新的父类seq container
  • 在seq container的play list中顺序拖入两个容器
  • 然后在seq container中把play mode改为continuous即可

Game Sync

啥是Game Sync

  • Wwise内置的用于接收游戏传来的消息的对象，例如之前的Event就是类似的一种
  • 同步游戏中和Wwise中的参数

Switch

常用于不同情况下的触发，离散参数的控制，脚步的触发等

创建Switch组接收游戏消息

  • 打开Game Sync选项卡后就可以新建一个Switch组
  • 选中一个Switch组又可以新建switch
  • 举个例子一个Switch组(Material)可以包含不同的Switch(sand,metal,concrete…)

创建Switch Container组织声音

  • 还是和以前一样导入文件夹，但是需要注意的是要将总文件夹设定为Switch Container，并且对每种材质对应的文件夹设定为random container

连接Switch Group和Switch Container

  • 在Switch Container的属性编辑器中我们可以为其选定连接的Switch Group
  • 以及一个默认的Switch，用于指定一个突发情况下的默认材质
  • 在Switch Container选项卡中可以为具体的Material赋值相应的footstep。直接将下方的容器拖入上方即可

创建一个event连接对应的Switch

  • 创建一个event
  • 将Switch Container拖入即可

Game Parameter(RTPC)

一定范围内连续变化的信息，发动机的转速，当日时辰，HP值等

创建

  • 大体步骤跟Switch基本相同，记得设定值域和默认参数

用Game Parameter调整属性

  • 选中SFX对象，在RTPC选项卡下选择x轴和y轴的参数
  • 然后我们就可以在下方的坐标图中绘制两个参数的曲线了！
在播放声音的同时移动坐标图的横坐标可以实时监听曲线对声音的影响
也可以在transport control视图下方便的调整RTPC值
  • Game Parameter可以映射至多个属性
  • 案例中我们除了用PlayerHealth调整音量，还控制了一个滤波器

State

从总体上去控制游戏的声音，对全局事件的响应，对玩家状态的识别

创建

  • 步骤还是大致相同，与Switch异曲同工
  • 先建立一个State组，再在其中添加新的状态

创建State间的过渡

State 有一个独特的功能，那就是它能向对象的属性施加一个偏置（offset），而且可以设定让这个偏置在一段给定时间内逐渐生效。比如，在主要场景为船上的游戏中，当玩家落入水中时，可借助低通滤波器来滤除游戏中几乎所有的声音。从水上状态转到水下状态的过渡时间可设为瞬时，从水下状态变为水上状态的过渡时间可设为几秒。对于后者，可逐渐减弱滤波器的影响，直到水从玩家耳朵中流干，再次听到所有频率的声音。

  • 只需双击State组，之后insert一个transition，两端对应两个状态，中间对应过渡时间

配置State变化时连接的SFX对象

  • 创建好State之后，我们需要将一个SFX对象与这个变化相连接
  • 玩家从活着到死去心跳声会逐渐减弱，我们就需要往HeartBeat这个SFX对象添加对应的State Group
  • 紧接着调整参数即可

同样的，我们可以在transport control视图中快速监听State变化带来的效果

appendix：显示voice volume的计算结果

  • 由于我们已经为HeartBeat这个SFX对象添加了很多的控制，有时候难以调试最终的结果
  • 根据文档设置好voice profiler之后，以后便可以很方便的查看是哪些因素在实时控制声音，以及声音的去向，音频流

营造空间

3D Specialization

利用位置信息来改变声音属性主要用到了attenuation曲线

attenuation曲线创建shareset

  • 创建attenuation曲线的前置工作，ShareSet是一种预设
  • 点开ShareSet选项栏后即可在attenuation文件夹下的某个work unit中创建attenuation

为对象指派Attenuation曲线

想象一个场景，游戏中每个传送门都会发出一个不间断的声音向周围传播。你需要离传送门多近才能听到声音呢？这就是Attenuation曲线解决的大多问题之一
  • 首先我们需要在SFX对象的positioning选项卡中将将 3D Spatialization 属性改为 Position + Orientation
  • 把下面Speaker Panning / 3D Spatialization Mix的值改为100的话就只会听到3D Specialization的声音
  • 然后在下方Attenuation选项卡中可以指派一个Attenuation，也就是我们之前创建的曲线

注意到 Attenuation Editor 跟你上一课中用过的 RTPC editor（实时参数控制编辑器）非常相似。它们本质上是非常类似的，但区别在于 Attenuation Editor 并不是把 Game Sync（游戏同步器）指派到特定的声音参数上去，而是自动把对象和 Listener 之间的距离指派到那些参数上。并且，有一组固定的参数可以受 Distance（距离）值影响。

**一定要注意距离值的设定！！！不然特别近才能听见

调整Attenuation曲线

Attenuation曲线与RTPC的调整其实大同小异

低通滤波器的小tips:
- 声音在空气中传播时，高频分量会被削弱，所以在距离拉远时，我们可以拉高低通滤波器的截止频率，可以达到这样的效果

添加Cone Attenuation(声锥衰减)

我们已经考虑了距离对声音的影响，那么当声源朝向其他方向时，我们如何兼顾这个因素呢？
  • 在Attenuation Setting界面的右下角可以看到一个圆盘，圆盘的中心代表声源，我们可以在这调节声锥衰减

音频信号流

在Actor-Mixer Hierarchy中组织声音

  • Actor-Mixer到底是啥？所有的参数是混合到总线之后再调整的吗？
  • 并非，Actor-Mixer实际上是对层级下的所有对象添加一个总体偏置
  • 例如，改变Actor-Mixer的音量(降低6dB)，那么会在这个层级下的所有对象当前的音量下减去6dB

使用 Master-Mixer Hierarchy

在这个部分完成所有声音的混音工作

Master Audio Bus(主音频总线)

  • 这条bus汇总了所有的声音
  • 唯一需要注意的就是其中的两个音量调节推子，一个是audio bus的volume，一个是选项卡下隶属voice下的volume，前者是真正地调节混合过的总线音量，后者还是添加偏置

其它音频总线进行子混音

需求：可能玩家想要单独地调节音量而不调节其它声音
于是可以创建一个新的音频总线去调节(在Master Audio Bus层级下)
  • 在创建之后，我们可以为对象指派发送到的音频总线
  • 在属性编辑器的routing选项下可以为其指派对应的总线

小trick：如果我们要一次为多个对象指派音频总线，只需要选中这些对象后Show in MultiEditor，然后在routing选项下从外部把对应的音频总线拖入即可

考虑一个情形：如果在一系列对象中有一个声效你不想把它指派到父类的总线中，那么我们可以在它的routing属性中把override parent选中即可

音频避让(Ducking)

  • 跟侧链差不多？

  • 不对，侧链是频率上的避让，这里指的更多是音量上的避让

  • 创建过程非常简单，只需要在对应音频总线的属性下找到Auto Ducking选项卡即可，下方可以选择用这条bus去影响哪条bus

插入效果器

  • 操作上非常简单，只需要找到对应的SFX对象，然后在Effects下添加效果即可
  • 要注意有些对象可能需要override parent
  • 需要注意的是最好打开render选项，这样可以不用实时应用效果器，减少系统开销

Auxiliary Sends(辅助发送)

作为附加输出，Auxiliary Send (Aux Send) 可由对象用于将音频发送到 Auxiliary Bus，并独立控制信号的发送音量。其中一个最常见的 Auxiliary Bus 应用场合就是为大量不同的声源应用各种效果（如混响或回声）。发送到 Auxiliary Bus 的信号会触发效果器，随后与未受效果器影响的干声信号相结合。

可以理解为Auxiliary Sends就是把音频信号分一份发送到一个轨道，在这个轨道上我们可以添加一定的效果，然后最终这个轨道会与干声混合出现在游戏中，我们只需要调整发送到这个轨道的音量就可以调节某种效果的影响程度
